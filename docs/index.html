Colors

The NSVRC styling is coded with SASS, so the following colors are specified using a dollar sign ($) followed by the name of the variable. Hover over the colors below to see their hexadecimal value.

Typography

The only font family used in the design is Proxima Nova, an Adobe Typekit font. The font weights used throughout the site include light, normal, bold, and black.

Videos

Thanks to Chris Coyier at CSS Tricks, videos can maintain their aspect ratio with a simple SASS mixin. Be sure to wrap your iframe or video element with divs using the classes video-container and content, and the aspect ratio you want in the @include mixin declaration.



<div class="video-container">
    <div class="content">
        <iframe src="https://www.youtube.com/embed/ScMzIvxBSi4" frameborder="0" allowfullscreen></iframe>
    </div>
</div>


Show SASS
.video-container {
    @include aspect-ratio(16, 9);

    iframe, video {
        width: 100%;
        height: 100%;
    }
}
@mixin aspect-ratio($width, $height) {
    position: relative;

    &:before {
        display: block;
        content: "";
        width: 100%;
        padding-top: ($height / $width) * 100%;
    }
    > .content {
        position: absolute;
        top: 0;
        left: 0;
        right: 0;
        bottom: 0;
    }
}

Side Navigation

The side navigation is displayed on interior pages and shows the secondary and tertiary navigation items. If the user is on a tertiary page, the entire tertiary navigation under its secondary navigation item will be open, as shown below. If the user is on a secondary page, its tertiary navigation items (if any) will still be shown. See the Top-Level page template for an example of this. Use the active class (documented at the end of the Links/Buttons section) on the page that the user is on to show where in the navigation they are. Also, the side navigation may or may not include the related story callout below it, depending on the page template.

Forms

When setting up a form for NSVRC, there are a few things to note. Namely, classes that will help you achieve the look you want.

  • full - Used on a div element wrapped around a label and input. This class makes the field full-width.
  • half - Used on a div element wrapped around a label and input. This class makes the field take up half of the total width.
  • half-container - This div technically doens't need to be named, as long as there is a container around all of the elements that have a class of half; otherwise the spacing of fields will be off.
  • required - This class is used on the label of an input that is required. It will add a red asterisk to the end of the label, visually indicating to the user that this is a required field.
  • error - This class is used on a field that contains an error after clicking the submit button. This will highlight the the field with a red border to show the user that this field needs attention.
  • validation-label - This class goes hand-in-hand with the error class. While the error class gives the field a border, the validation-label class will color the validation message in red. This class is used on a p element that will show when the user has an error in the form, and will display the appropriate message.

Here is an example of a form that includes all of the classes mentioned above:

<form>
    <div class="full">
        <label for="name" class="required">Name</label><input type="text" name="name" id="name" required />
    </div>
    <div class="half-container">
        <div class="half">
            <label for="email" class="required">Email Address</label><input type="email" name="email" id="email" class="error" required />
            <p class="validation-label">Email Address is a required field</p>
        </div>
    </div>
</form>

Calendar

The NSVRC calendar uses a combination of width, float, and a jQuery plugin called matchHeight.js to create a flexible grid. Like the forms, when setting up the NSVRC calendar, there are some classes to note that will help you achieve the correct look.

  • unselectable - This class grays out the calendar date, inidicating that this day is not from the current month.
  • two-days - This class is used alongside the event class. The two-days class stretches the event to accomodate two calendar days. Additionally, there are class names to accomodate all types of event lengths. These classes are three-days, four-days, five-days, six-days, and seven-days.
  • has-event - This class isn't evident on desktop, but it's necessary for the calendar's mobile styles to appear correctly. Use this class in conjunction with the cal-date class for every date that an event is going on, and the mobile calendar will show a gold background on those days.
  • mobile-date - This class is used on a div element and appears before an event's link (on the mobile calendar only). The div specifies the date or date span of the event in the format 00/00/00, while the class itself styles it with a gold background.
  • mobile-cal - This class wraps around the mobile-date and event classes to create one mobile event, separating if from other events with a gray line (on the mobile calendar only). So if there are two events on the same day, they would each need a container with a class of mobile-cal in order for them to show as separate events.

Header

NSVRC's header is made up of several parts: the social media bar, the logo, a CTA nav, main navigation (including the secondary and tertiary navigations on hover or click), and a search bar. All of these elements are included in both the desktop and mobile version of the header. You can see the header in context on any of the templates.

An important thing to note about the main navigation is that when a user is on a page, that page should be highlighted in the main navigation. For desktop, use the class current on the page's top-level parent navigation item. This will change the background color of either Find Resources, Find Events or Our Work to dark gray. For mobile, the current class should not be used. Instead, use the active class on the exact page the user is on. So if they are on the Resource Topics page, only that page should have the active class. To see this in action, resize the browser window and look below to see how the header changes.

Another thing to note is that second-level navigation items that contain their own subnavigation should have the class has-child (desktop nav only). This will give those items a right angle icon, indicating that there is a subnav.

In the mobile nav, add the class openClass to the active class' "breadcrumbs". This will keep the mobile navigation open so the user doesn't have to tap several times to see other pages in the subnav they're already in. For example, to show the subnav that the "Resource Topics" page is in by default, use openClass on the li elements for "Find Resources" and "Advocates & Educators".

Also note that since secondary navigation items link to their own pages, on the mobile version of the header there will be duplicates of them - one that the user taps on to see its subpages, and one that actually links to the page. For example, if you tap Survivors in the mobile nav, the link directly below it will also say Survivors. This is the actual link to the Survivors page.

Enhanced Safety Planning for Immigrants Survivors of Domestic and Sexual Violence
  • ASISTA
  • 2017

Abusers often use the threat of immigration enforcement as a way to maintain power and control and to make victims less likely to seek protection. For this reason, it is important for advocates to understand how to: help immigrant survivors become aware of their rights; identify special immigration remedies for victims, including special VAWA provisions around confidentiality; and how to prepare enhanced safety plans for immigrant survivors that take into account immigration enforcement.

Primary Prevention Primer
  • NSVRC
  • 2016

This interactive learning tool introduces primary prevention through a set of activities to enhance knowledge about primary prevention. Information covered includes the Moving Upstream story, the Social Ecological Model, and an exploration of the differences between Risk Reduction and Primary Prevention.

Campus Public Safety Online Series
  • The National Center for Campus Public Safety
  • 2016

The National Center’s college campus safety webinars page Campus Public Safety Online, includes upcoming webinars and links to previously conducted, closed captioned webinar recordings. Webinars cover topics of interest to the college campus public safety and emergency management community including fair and impartial policing, trauma-informed sexual assault investigations, large event management, National Incident Management System, behavioral threat assessment and others.
